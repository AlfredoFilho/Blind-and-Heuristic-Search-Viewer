About Search Algorithms Visualizer

Project Overview

This web application provides a visual representation of several common pathfinding search algorithms. It's designed to help understand how these algorithms explore a grid to find a path from a starting point to an ending point, while avoiding obstacles.

Algorithms Implemented

  • Breadth-First Search (BFS)

    BFS is an algorithm for traversing or searching tree or graph data structures. It starts at the tree root (or some arbitrary node of a graph, sometimes referred to as a 'search key') and explores all of the neighbor nodes at the present depth prior to moving on to the nodes at the next depth level. It is commonly used to find the shortest path in unweighted graphs.

  • Depth-First Search (DFS)

    DFS is an algorithm for traversing or searching tree or graph data structures. The algorithm starts at the root (or arbitrarily chosen node) and explores as far as possible along each branch before backtracking. It is often used for finding connected components.

  • Best-First Search

    Best-First Search is a greedy search algorithm that explores a graph by expanding the most promising node chosen according to a specified rule. It uses a heuristic function to estimate the cost from the current node to the goal.

  • A* Search

    A* Search is an informed search algorithm that finds the shortest path between a starting and a target node in a weighted graph. It combines the features of Dijkstra's algorithm (which guarantees the shortest path) and greedy best-first search (which is fast but not guaranteed to find the shortest path). A* uses a heuristic function to estimate the distance to the goal, and a cost function to track the actual cost from the start node.

Development

This project was developed using HTML, CSS, and JavaScript. The visualization is rendered on an HTML5 Canvas. Each search algorithm is implemented in a separate JavaScript module for clarity and maintainability.
